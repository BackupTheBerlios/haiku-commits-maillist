> OK in this case the ternary operator is not appropriate, but in

It is allowed and encouraged in expressions, at least when it makes the 
code more compact and readable (definitely a matter of personal 
preferences). E.g.:

	BView* parent = (view ? view->Parent() : NULL);

or

	return (error == B_OK ? bytesRead : error);

In both cases using "if" constructs is OK, too.
