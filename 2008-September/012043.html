> I'm not sure if that's a really good fix. It's like opening a warning 
> alert on the first startup that all GPL add-ons are missing due to 
> license issues; I think it's okay to mention these kind of things in some 
> accompanying document, but I don't really like to put stuff like that in

But that is exactly what you have to do to circumvent the supposed patent 
issues. If you live in a country which is not affected...

The funny thing is that Haiku has global subpixel based anti-aliasing 
regardless of the freetype code being compiled in or not. Albeit with our 
own "filtering". What gets left out in freetype is the rendering of glyphs 
three times as wide - only that much. I can't really believe that this 
portion of the subpixel text rendering is actually patentable...

If we wanted to be cautious, we could completely disable even the 
compilation of our own subpixel stuff according to the freetype option, so 
that it is enabled in one single place.

Even then, the hinting on versus off issue remains. Some people like blurry 
font rendering which is more truthful to the original shapes in the font 
file.
