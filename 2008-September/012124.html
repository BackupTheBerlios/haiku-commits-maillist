An invalid value for bytes per row should actually be anything that 
cannot really take the width of the bitmap. I've changed it now so that 
values > 0 and B_ANY_BYTES_PER_ROW are accepted. While this is not 
entirely correct, as it doesn't catch a too small bytes per row value, 
it should be acceptable, as really calculating that would probably be 
needless overhead. Thanks for spotting though!
