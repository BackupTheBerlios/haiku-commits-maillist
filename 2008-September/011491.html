Ouch...

I have ported exactly that driver some more than a month ago. My version
doesn't crash, however sending doesn't work. It seems that there is a problem
with our emulation layer as it seems other FreeBSD drivers aren't working
either (there's a VIA rhine bugreport mentioning similar symptoms). I haven't
commited it as Stas (who wrote the driver) mentioned in some post that the
version is outdated and that a newer version would be made available. Since it
isn't usable as is I decided to wait for the update first to check whether
this would get it going.

BTW I used the atphy instead of the ukphy and added the F2 to that one. It
seemed to work better in some regard, though sending wasn't affected by that.
Not sure it would actually work in the end.
