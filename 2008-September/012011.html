> 2) a default format (like PNG) should be preselected
There will be some kind of small settings, like the last five 
locations saved to and the last saved outputformat + presetting 
the first time started. As you wrote, it's work in progress.

> 3) I don't really like the pretty unusual (interface-wise) "Options" 
> page. Since the "screenshot delay" doesn't seem very useful to me (why 
> not just press the key later?), one possible option would be a single 
> options menu containing all items.
I'm still a bit unhappy with how it's right now, so I'm always up 
for better ideas.

But menus are a no go for me. I don't like the way this works to 
set something, see e.g Deskbars options. To simply turn on 
seconds, euro date, expando or whatever one has to perform a 
bunch of off steps and start over and over again. Popup menus are 
even worse, e.g ActivityMonitor is kind of frustrating to turn on 
some options there. Even if these actions are performed only once 
it could be done better.

I think all people involved in Haiku are great developers but 
that does not instantly mean we know how to properly design a 
usable application (UI wise). These are jobs for UI designers, 
which seems we kind a lack off.

> 4) I don't like the separator bar too much, generally; I would only add 
> it when the window contents are confusing else, ie. to separate 
> functionality that doesn't fit together well (and then, a named box is 
> alsmost always the better option IMO).

I think I'm going to re-group that a bit, getting rid of the bold 
Options thing at least.
