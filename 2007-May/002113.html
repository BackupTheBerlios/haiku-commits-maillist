See r21020 as an API proposal, feel free to comment. In the current 
implementation, B_WATCH_RECURSIVELY is ignored (it always works 
recursively right now, and maybe there is little reason to include this 
flag).

It sends B_PATH_MONITOR messages that have the same fields as the usual 
node monitor message, plus "added" and "removed" in case files were 
added or removed into the watched space (I currently don't test if the 
formerly known parent directory is actually contained in the search 
path).

API :-)
