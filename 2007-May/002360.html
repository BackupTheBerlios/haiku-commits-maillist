> > Fixed the problem of not being able to work from Haiku. The main problem was that although the multi_audio API on BeOS hands the driver the settings as made in the Media prefs, the Haiku one just gives maximum values and assumes the driver will scale the values down. I didn't do that.
>
> Maybe we should fix our API to mimic what BeOS does, or is this
> intentional ? Jerome ? Marcus ?

IMO multi_audio addon does the expected behavior. B_MULTI_GET_BUFFERS
is used to get buffers, not to set them. The request_playback_buffers
and request_record_buffers parameters are used to evaluate if the
playback_buffers and record_buffers are big enough to hold the buffer
description, hence the maximum values. Also, for other drivers,
buffers are not set up on the B_MULTI_GET_BUFFERS call, but as part of
the initialization phase.
B_MULTI_SET_BUFFERS is not used as of now.
