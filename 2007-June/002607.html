I once tried to replace the Application menu in Zeta with queries on an 
indexed string attribute named APP:group, like:
'(APP:group=="*[Dd]evelopment*")'
'(APP:group=="*[Ee]mulator*")'

Someone wanting to keep a single Applications menu could use something 
like:
'(APP:group!="*[De]mo*"&&APP:group!="*[Pp]reference*")'

This would suppress the need to add the symlinks in the app installer, 
which usually always end up at the wrong place.

Of course it's not as flexible for those who want only specific things 
in the menu, but one could always have his own symlinks alongside in 
another folder.

Also needs to be limited to the boot volume somehow, don't want to end 
up with 10 times Terminal in it... But the query files use a flattened 
BMessage with the volume sig for now for target volume, which isn't 
known at build time.
