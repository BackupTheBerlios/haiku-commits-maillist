Hmm that's a nice idea, but instead of having multiple 
B_(EXTRA_)[n]_LIGHT flags (you'd need the _HEAVY versions too anyway, 
and it's an uint16!) I'd rather have B_SERIF and B_FANTASY or whatever 
do describe non-regular-looking ones. B_SCRIPT maybe ?
I rarely that face thing, I try to always use family and style, like in 
XEmacs, which uses a string descriptor including size anyway, but for 
NetSurf I had to map flag-like constants for weight and others.

The weight thing seems to be more of a float value by itself on other 
platforms though, like Pango and RiscOS.
See:
http://source.netsurf-browser.org/trunk/netsurf/gtk/font_pango.c?revision=4033&view=markup
http://source.netsurf-browser.org/trunk/netsurf/css/css_enums?revision=2505&view=markup
