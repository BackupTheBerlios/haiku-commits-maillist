I usually find code more readable that does have as little extra 
characters as possible; if someone uses parenthesis when they aren't 
needed, I always think there must be a reason for this which costs 
extra time parsing it :-)
Plus, it's always good to know the operator precedence rules to read 
other people's code, too :)

For example, I'm really annoyed by that GCC4 "feature" that it warns 
you to use an extra set of parenthesis in a case like this: "a || b && 
c" when this is clearly not necessary given the operator precedence 
rules (you need to write it as "a || (b && c)" to get rid of that 
warning).
I would all be for removing that warning if that's easily possible.
