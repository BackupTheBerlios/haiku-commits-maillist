If the downloaded file has the same name as the file being downloaded
(in this case, looks like Firefox.zip) it will not overwrite it. You
will have to delete Firefox.zip in your generated/downloads directory
and re-run.

To prevent this type of confusion, it might be safer to use the full
filename locally for the downloads as such:

InstallOptionalHaikuImagePackage Firefox-2.0.0.12
: http://www.yellowbites.com/downloads/Firefox-2.0.0.12.zip
: apps
;

I usually end up just adding rules to my UserBuildConfig to unzip and
create the symlink for one of the new experimental builds anyway.
