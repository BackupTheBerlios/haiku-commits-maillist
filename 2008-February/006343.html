If the file is a script our execl(), execv(), execle(), and execve() 
functions prepend the interpreter line to the argument vector, but leave it 
otherwise untouched. load_image() doesn't have a separate executable path, 
so argv[0] is always the file/path name. Our exec*p() are still not 
correctly implemented, BTW.
