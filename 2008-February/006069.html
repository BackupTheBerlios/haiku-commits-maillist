Well, it's certainly a nice milestone (if we even want to call it that), but I
don't think we need to hype it. Haiku's just become pretty stable and nice to
work/develop on. But it's certainly not there yet. As I said there are
numerous memory leaks (or maybe it's just the new allocator who messes up
things...) that prevent doing everything in one go. So we're not exactly
selfhosting, just close to it.

Anyway, using the R5 buildtools is not really what we aim at. The toolchain
will be ported to Haiku natively so it can be installed without removing R5
specifics from a compiler that does things not really necessary for Haiku.
Only with native buildtools and when we reach a point where we can do a full
cycle without having to reboot we can start calling it selfhosting.
