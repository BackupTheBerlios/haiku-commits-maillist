"default" counts as a label, too. For the same reason: The variable is 
declared and has an initializer, but the initialization is skipped when 
jumping to the default case. The following test doesn't compile, for 
instance, even if the compiler could deduce that the "default" case is 
never jumped to:

int
main()
{
	const int i = 0;
	switch (i) {
		case 0:
			int t = i;
			return t;
		default:
			return 42;
	}
}

Splitting declaration and initialization (i.e. turning the initialization 
into an assignment) helps BTW:

int
main()
{
	const int i = 0;
	switch (i) {
		case 0:
			int t;
			t = i;
			return t;
		default:
			return 42;
	}
}
