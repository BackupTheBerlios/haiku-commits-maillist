wrote:
> Marcus Overhagen <marcusoverhagen at arcor.de> wrote:
> > bonefish at BerliOS <bonefish at mail.berlios.de> wrote:
> > > +extern "C" status_t
> > > +is_vnode_removed(mount_id mountID, vnode_id vnodeID)
> > 
> > > +    if (vnode)
> > > +        result = vnode->remove ? 1 : 0;
> > > +    else
> > > +        result = B_BAD_VALUE;
> > That mix of bool and status_t is really crude, especially because
> > false == B_OK.
> > Wouldn't it be enough to return a bool only, with  B_BAD_VALUE = true
> > ?
> > 
> > or make it
> > 
> > status_t is_vnode_removed(mount_id mountID, vnode_id vnodeID, bool *

> Indeed, there is little reason to keep that thing BeOS compatible.

I find it quite consistent with other cases where an alternative error 
code/value is returned (e.g. create_{sem,port}(), ssize_t for reads/writes, 
FDs) -- a value < 0 is an error code, >= 0 means an actual value of the 
respective domain. B_OK is not a valid result (which is why I find the 
check < B_OK not correct in those cases, BTW).

But anyway, I'll changed the function according to Marcus' second proposal, 
renaming it get_vnode_removed() then, though.
