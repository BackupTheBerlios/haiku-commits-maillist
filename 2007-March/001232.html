It might be weird, but I don't think its a bug. There's simply a difference 
between initialization and assignment. In case of C++ classes the 
constructor is used for initialization while the assignment operator is 
used for assignment. Analogously in a switch statement the jump to a later 
case label would skip the invocation of the constructor, thus leaving the 
object in an illegal state. For primitive and C types initialization is 
optional, which is why the second version works. Might be weird, but it is 
nevertheless pretty consequent. :-)
