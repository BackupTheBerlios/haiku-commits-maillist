On Wed, Nov 26, 2008 at 12:05 PM, oruizdorantes at BerliOS <
oruizdorantes at mail.berlios.de> wrote:

> -Add the possibility to post formatted messages to the debug window
> -Make the window resizable. BUT as this is a multitab window when I resize,
> only the view in the active TAB is resized, as soon as I switch to another
> TAB the view in the new selected tab has its old size, messing all.
>

That's a problem in BTabView, and unfortunately we have to keep that
behavior for R5 compatibility reasons, at least when using the normal
constructors (R5's BTabView detached the not-visible children when switching
tabs, so they had no way of getting resize messages that happened when not
visible). You can avoid it if you use the layout-friendly version of
BTabView, but that will also require using the layouts in general, which I
wouldn't be able to give you the best primer on. Ryan's written a nice
introduction on it here:

http://www.haiku-os.org/documents/dev/laying_it_all_out_part_1

Hope that helps,

Rene
