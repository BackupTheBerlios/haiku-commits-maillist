Jam can indeed find additional header dependencies, but only those that are 
in the search path, too, which in this case really shouldn't happen. Jam 
searches headers in the directories specified by the per object variable 
HDRSEARCH (cf. the Object rule in build/jam/OverriddenJamRules), but those 
should be identical to the search paths passed to the compiler.
