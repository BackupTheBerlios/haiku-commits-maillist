Stephan Assmus <superstippi at gmx.de> wrote:
> One more thing that pops into my mind: atomic_XX() take volatile ints 
> for a 
> reason. Will everything work like it is supposed to if one just casts 
> some 
> memory buffer offset to vint32?

The volatile attribute only means that the compiler won't cache the 
value in a register for faster access, but always references it from 
its memory location. As long you don't access the value directly, you 
don't need to specify it (ie. if you only use atomic functions).
