Indeed I used the 5.8.0 version of perl to get autoconf going. Anyway I 
have now built a package of the cross tools that works on BONE, haven't 
and cannot test on plain R5 though. I built it using "configure --build
-cross-tools" from "/boot/apps/haiku" and zipped up the cross-tools 
folder. The zip is availalbe from http://haiku.mlotz.ch/haiku_cross_compiler_bone_2.95.3-haiku-080323.zip 
and should be extracted to "/boot". I successfully configured another 
machine with "configure --cross-tools-prefix /boot/apps/haiku/cross-
tools/bin/i586-pc-haiku-" and built a working haiku.image with that.
So it would be nice if someone could try that under plain R5 and maybe 
we could upload that package to haiku-files and put a link with 
instructions onto the developer page.
