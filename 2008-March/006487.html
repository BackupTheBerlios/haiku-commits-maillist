It seems that this broke the build for BONE. I had to add the following 
to the libhaikucompat jamfile to get it to work again over here:

if $(HOST_PLATFORM) = bone {
	ObjectDefines $(libbePatchSources) :
		USING_HAIKU_TYPE_CONSTANTS_H= ;
}

Which is the same as the one for Dano except for the "IMPEX_BE=" part 
as this generates a lot of redefinition warnings under BONE.
