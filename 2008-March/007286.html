It adds another variable to how the image is created, which, at least 
theoretically, might influence behavioral aspects (our runtime loader even 
reads the gcc version from the executables, although it ignores the 
haiku/beos part ATM). So I'd rather not do that. If someone is really eager 
to use the BeOS compiler, they can still just uncomment the version check 
in "configure".
