To make this class truely thread-safe, we would only need to make the 
detach process safe. That will probably require some locking, but would 
therefore also make the memory footprint a bit higher.
When using benaphores, the performance is probably not that bad.
Another way would be to have all threads go through a single lock on 
detach, but depending on the usage pattern, this potentially degrades 
performance a lot.
