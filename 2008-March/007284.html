Well, using TLS, as your implementation does, would also help with files. The only 
thing more robust with one file per entry is when a program iterates through the DB 
just when it is changed. The single file version could read partially invalid data in 
that case, but that is pretty unlikely and easily avoided by writing to a different 
file first and rename()ing it to the actual one when done.
