> If I understand correctly, the "GCC compiler 2.95.3 (for BeOS 5.0 or
> later)" linked from the website and haiku-files.org cannot be used to
> build Haiku anymore. If that's the case, I think we should remove it.
> Please confirm it is OK to do so.

That's correct, it cannot be used to build Haiku anymore. But there are the
buildtools that are built for the host platform (bfs_shell, makebootable and
the like) that still need a working host compiler. I don't know if the plain
old version included in the R5 development tools is enough or if we still need
the updated one. Generally I wouldn't remove it from there just yet.

> Additionally, it sounds like people could benefit from the cross
> compiler binary built by Michael. Is it OK to link to this binary from
> the Downloads/Development pages of the website as well as
> haiku-files.org?

Yes, I guess it should work for most people. Feel free to link it from those
pages. Best would be if you could also state that you need to unzip to "/boot"
and that you need to use "configure --cross-tools-prefix
/boot/apps/haiku/cross-tools/bin/i586-pc-haiku-" to get them going.
