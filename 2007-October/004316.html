Some input from my side: After reading the introduction I see that you focus
heavily on the message delivery and the BMessage used as a real "messages".
While this is certainly one central point of BMessages they are, from an
technical point of view, mostly storage containers which enable easy and quick
access to named and typed fields. A common usage of BMessages which is often
forgotten for example is to save settings files or archives of some sort. If
I'm not mistaken also some file formats (WonderBrush?) are based on BMessage.

Otherwise it looks good (as far as I have read). Notify me when you need me to
review the document. I know that I'm still lacking behind on the USB
documentation review and I'm sorry for that BTW.
