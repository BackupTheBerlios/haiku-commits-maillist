You have to log in locally, i.e. from within Haiku. When connecting from 
another host, errors occur only occasionally.

> Anyway, how would I run the OpenSSH tests best? (assuming I have a bare
> Haiku image) :-)

* Enable OpenSSL optional package.
* Get the OpenSSH 5.0p1 sources.
* Configure with:
	./configure CPPFLAGS=-I/boot/develop/headers/bsd LIBS=-lbsd 
--prefix=/boot/home/config --with-ssl-dir=/boot/home/config
* make
* make test

The first test with the connection problem is "login-timeout". To skip the 
tests before edit the LTESTS variable in regress/Makefile.

Unrelated: If you want to see Haiku crash in the slab allocator or net 
buffer code, try the "forwarding" test.
