> > That change is actually not correct, though, as the system will now
> > return B_NOT_A_DIRECTORY when in fact it is a directory with missing
> > privileges.
> > Correct would be to check if "vnode" is a directory beforehand -
> > probably in the calling function.

> Yes, but I did not see any obvious way to check if a vnode is a

The VFS can always read_stat() a vnode to find out. Moreover lookup() also 
returns the type, so at least from the second iteration on we already do 
have the type of the supposed directory node. So only passing the type of 
the directory node to vnode_path_to_vnode() (for the first iteration) will 
be necessary. That needs some checking whether/when that type is always 
know and get it, if not.
