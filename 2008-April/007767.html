On 2008-04-14 at 16:31:20 [+0200], Rene Gollent <anevilyak at gmail.com> wrote:
> With respect to the read-only case
> that started this discussion though, wouldn't it potentially be
> simpler to mount config/settings as a memfs if that case is detected?

Although it probably still needs a bit of work, my recent changes to the 
file system interface already laid the groundwork for adding layers on top 
of mounted file system. So it should be possible to implement a layer that 
adds write support to a read-only volume by storing the changes in memory. 
When the kernel mounts the boot volume and finds it read-only, it would 
simply put that layer on top of it.

With the same technique an attribute/query support layer could add 
attribute/query support to a standard ISO9660 CD (the additional data could 
be stored in special folders). This would finally allow us to create 
bootable single-track Haiku CDs.

CU, Ingo
