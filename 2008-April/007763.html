"Rene Gollent" <anevilyak at gmail.com> wrote:
> I just thought it seemed like a simpler solution since otherwise 
> every
> pref has to have special case code for read-only, which seems 
> wasteful
> if it's only ever going to be used for the LiveCD anyways, which in
> theory would only be used once for a little poking around at which
> point the user would likely install to disk or pitch it anyways.

There are other points besides that makes working in that area a good 
idea, namely:
* no file locking is done - a keymap could be invalid while the 
input_server reads it
* two apps need to know how the keymap files are encoded, and need to 
know how to retrieve them (ie. prepare for one setting per keyboard vs. 
one setting for all)
* the input_server will need to be more flexible with its settings 
anyway for things like multi-user
* support for changing keymaps on the fly (ie. via a Deskbar add-on or 
shortcut)
