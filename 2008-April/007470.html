While it's nice to have a downloadable dev package, and those might not 
be needed for a base distro, it's still useful to have the sources 
around IMO.
First, because building a distribution is much easier if everything is 
cross-buildable. It might be harder to get right than running configure 
scripts and make, but I already went the other way once with Zeta, 
writing a makefile engine to run configure and make install apps from 
the source tree to get the full distro, but it's not reliable.
Also, some platforms (m68k, but later arm and even ppc) might not even 
have enough resources to be able to natively build parts of those. 
Using qemu-ppc to build them is not really a solution.

François.
