I thought of that too, but Axel wanted it moved client side like on
BeOS. Plus it allows for code like you had in MediaPlayer to "prepare"
for a screenshot by turning off overlay, etc.

>  This is very dangerous. Any 3rd party application might "update" libz.so
>  or libpng.so and make libbe.so fail to load. libbe.so should not have any
>  dependencies except libroot.so, and libroot.so shouldn't have any dependency.

>

I think the overall consensus is to move the screenshot logic to an
external application, which I think is fine.

The only reason I kept the PNG screenshots was because that is what
app_server did before. I still think it is useful, but would be better
done with the Translation Kit. I think that PNGDump code was just
there because we didn't want the app_server using the Translation Kit.
I just copied it since it seemed easiest, but in retrospect we need
something better.
