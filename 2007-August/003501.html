> That reminds me that the debug server does not really like spaces in
> the path to the executable (or its name). I should probably just open a
> bug report for this, though :-)

Not for the debug server at least, since it's not to blame. It passes the 
correct arguments to the Terminal. The Terminal joins the argument vector 
to a single string in ArgvReceived() and later (in Shell:_Spawn()) 
disassembles the string again (splitting it at whitespaces). Even if the 
debug server would escape the argument vector strings -- which makes no 
sense, since they never goes through a shell -- it would still not work.

Either the Terminal has to escape the argument vector elements when joining 
them in ArgvReceived() and deescape in Shell::_Spawn(). Or it has to pass 
argument vectors around.

CU, Ingo
