Jonas Sundström wrote:
> Could Haiku know itself what style of keyboard
> is present and its keymap? 

I hope it could, but our HID driver would not support that yet, anyway 
(as we only use the boot protocol).

> (PS2/USB - any difference in what is possible 
> to find out about the keyboard?)

PS/2 has no possibility of this at all, but USB has some.

> MacOS X has a user dialog at install time asking
> the user to press the key directly to the right of
> the left-side shift key, IIRC, giving the system
> some rudimentary idea which of 3 major types of
> keyboards the current one is.

I came across that one as well; it would be a possibility for PS/2 
keyboards, but ultimately, it doesn't matter that much, anyway -- they 
keymap isn't changed in any way on Haiku, it's just for the looks in 
Keymap.

> For Haiku this dialog could happen when opening
> the  Keymap preferences the first time. (if its
> not possible to probe a keyboard silently)

I'm not sure yet how that will work, but I plan to have per keyboard 
keymaps. It might be solved similarly to what the app_server currently 
does for monitors (ie. use whatever the keyboard reports, remember 
model and product number to recognize similar keyboards that share the 
same settings, but can still have different ones).
