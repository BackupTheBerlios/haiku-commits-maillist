> Clearing this bit fixes DMA transfers for me.

Turns out, DMA transfers are still not working. They are disabled after 3 failures,
the system reverts to PIO, and boots into the desktop.

However, whatever we do, we *must* clear the PCI BMDMA interrupt bit,
and must read the device status register (not altstatus) both *inside* the 
interrupt handler.

Doing it later won't work on many systems, because the controller keeps 
firing interrupts, preventing the finish_dma() and similar code from beeing
executed.

I'll look into this later this week.
