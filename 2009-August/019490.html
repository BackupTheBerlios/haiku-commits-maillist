> If we need such (dirty) tricks, I would say our API is broken in this
> regard.
> Ingo, isn't there any way (yet) to make sure the window is layouted at
> a certain point to simplify this?

The initial layout happens asynchronously when the window looper thread 
processes the B_LAYOUT_WINDOW, but you can call {Min,Max,Preferred}Size() 
on the window's layout directly after constructing the view hierarchy and 
set the window size to whatever you like best.

Note that the window only changes the size it has been constructed with, if 
the B_AUTO_UPDATE_SIZE_LIMITS flag has been specified and if the size is 
out of the min/max size range.
