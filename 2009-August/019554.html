> Michael Pfeiffer <michael.w.pfeiffer at gmail.com> wrote:
>>> For now copied PrintTransportAddOn.cpp from kits/print into each
>>> tranport add-on folder and use that instead.
>> Is there a way to reference the file PrintTransportAddOn.cpp from
>> folder .../kits/print?
>
> Sure, there are at least 3 possibilities:
>
> 1) (after the sources)
> SEARCH on [ FGristFiles PrintTransportAddOn.cpp ]
> 	= [ FDirName $(HAIKU_TOP) src kits print ] ;
>
> 2) (before the sources)
> SEARCH_SOURCE += [ FDirName $(HAIKU_TOP) src kits print ] ;
>
> 3) make a static library and include them from both.

I had (3) before I made the change and it did not work,
I think because the library had a dependency to a function
defined in the printer transport folder (and not in the
other direction), so the linker did not include the symbols
from the static library (containing PrinterTransportAddOn.cpp).
(PrinterTransportAddOn calls the function instantiate_transport()
which is for example defined in ParallelTransport.cpp,
SerialTransport.cpp and USBTransport.cpp).

I am going to change it to (2).
