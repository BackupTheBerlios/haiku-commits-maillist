> IA-32 Architectures Software Developer's Manual, Volume 3A). It seems 
> these 
> days (for a while even) one simply can't rely on the ordering of

That's indeed true for a number of years - besides, other CPU 
architectures are using out of order memory accesses even longer, so 
even if code like this would work on x86, it would not be portable.

functions, and you should be fine.
I am not sure we need anything beyond that, although it surely wouldn't 
hurt to add some.
