libroot.so is, though. It uses versioning for the *stat() functions, which 
are pretty likely used in gcc. When any versioned symbol is used, the 
compiler/linker generates a version table for all symbols. But that's only 4 
bytes per symbol, so it shouldn't add dramatically to the executable size.
