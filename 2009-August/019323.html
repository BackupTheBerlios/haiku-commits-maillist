> I'd prefer a simple beep - alerts like this tend to annoy more than

Totally, though I also hate beeping.

I think the way most browsers do find these days is quite nice (not
that we want to complete redo all our find interfaces right now but it
is something to consider for the future.) Chrome as one nice example
adds a little widget to the top left of the content area that let's
you search and then shows the number of results. If there are no
results the number of results is 0. Arrow buttons then let you go back
and forth to each result. Simple, unobtrusive, no annoying dialogs or
beeping. I will probably copy this in our browser.

Finally for long term code reuse and consistency maybe we should put
some generic find dialog code in the shared code instead of having it
done slightly differently in various apps.
