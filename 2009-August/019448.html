> I was not talking about an overlay system to allow live queries on 
> non-BFS volumes (although I would not mind that capability). What I 
> meant was a system that would retain the ability to use filenames 
> without extensions in Haiku, with the OS itself adding the extensions 
> when those files are copied/moved to a non-BFS volume (or viceversa).

It can be done in Tracker - easily even - if one ignores
the other points where data enters and exits the system.

But assuming that people either want or do not want filename
extensions is naive. Here are some issues that would have to
be adressed somehow:

I may like for some filetypes to lose their extensions when
I bring them onto my Haiku system, but I want other files to
keep their extensions. They might be non-essential but I like
to have them, or maybe they're essential even in Haiku. 
(e.g. source code)

I might -sometimes- need to do file transfer where absolutely
no filename changes occur, whereas most of the time I don't
care. E.g. moving files from a Linux system, across Haiku, to
a Windows system, with no extensions lost that were there and
none added that weren't there.

I may like for some filetype to get a filename extension when
I move it to a -certain- non-BFS filesystem but not when I move
it to some other such filesystem.

> How to implement this cleanly and without overdoing it, I 
> don't have a clue. That's for the engineers to figure out. :)

Well, I don't have a clue either.

/Jonas.
