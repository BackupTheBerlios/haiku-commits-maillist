BTW I will probably start working on a Ruby script to help make all
this easier. If you think you might find it useful let me know and
I'll get it going sooner. The basic idea would be to use the svn log
xml syntax to find all the changes with the +alphabranch since the
last merge, and then merge those into the alpha branch working copy.
The changes can then be audited by hand, compiled and tested before
committing. The script could keep track of the last merged revision
(which of course will also be in the commit message), so future
merging is easier.

Haikuware has a fairly recent Ruby for Haiku, BTW. I'll probably be
working in Linux though.
