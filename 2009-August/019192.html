- AFAIK, Haiku's DriveSetup *can* setup partition (create, delete),
but only primary partitions. Extended partition support is not (yet)
mature enough. But maybe I'm wrong here.

- Beside the sample settings to add to GRUB menu.lst, would be great
to add a link to GRUB manual
(http://www.gnu.org/software/grub/manual/grub.html) so we don't assume
an Haiku user is also a Linux uber-nerd who know absolutely all about
GRUB setup...

- AFAIK again, Write Boot Sector button is an option to just do that,
installing Haiku's boot loader into boot sector. A full install does
it too, and user won't need to do it after a full install, but only if
he need to reinstall it (after he have installed another OS and
discovered that his usual lovely Haiku Boot Loader was replaced by
Microsoft, or... by a Linux distro GRUB, without asking. Booting from
a bootable Haiku CD or USB key, this option allow him to reinstall
Haiku's bootloader, without reinstalling all.

Maybe the button label should make it more clear. Like "Setup boot
loader", calling bootman ?
Or maybe, once again, I'm totally wrong here too... :-)
