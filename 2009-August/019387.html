Hi Urias,

Urias McCullough wrote:
> On Sat, Aug 15, 2009 at 10:01 PM, Jorge G. Mare<koki at haikuzone.net> wrote:
>   
>>> This was absolutely annoying...
>>>       
>> Why is it annoying?
>>     
>
> Because most of the screenshots attached to Trac are non-viewable in
> the browser.
>   

That seems to be the wrong reason for making a change like this.

>>> and we have to interface with
>>> windows/linux users for the foreseeable future.
>>>       
>> So? Are we then going to start adding extensions to the other files
>> types too? Perhaps we should also throw away BFS attributes so that we
>> can send People files to Window or Linux users?
>>     
>
> Keep in mind, we're talking about screenshots here... not every file
> created on Haiku. If I made a file with the express intention to send
> to someone else, I'd certainly want an extension added to it so they
> had half a clue what I was sending them. That's pretty much the sole
> purpose of screenshots IMO.
>   

There is no way to draw the line IMO. Doing this here sets the precedent 
for eventually doing this for all other files types; once you open the 
floodgates, that's what's going to happen. People will then say: oh, 
this is necessary because mp3 players need it, or because other OSes 
don't <insert other OS braindeadness of choice here>, etc. Before you 
know it, Haiku will have inherited the braindeadness of other OSes, 
making the whole argument that we don't need file extensions because we 
make intelligent use of mime types a moot point.

>> The "Windows is so dumb that needs extensions to know the file type"
>> joke has always been a classic in the BeOS/Haiku scene, and now we make
>> Haiku behave the same way, unnecessarily and for the wrong reasons.
>>     
>
> Not just windows, but every browser that has to show a screenshot
> where the server doesn't provide the mime type...
>
> Also half a dozen other OSes that also have no mime-sniffing magic in them.
>   

My question to this is: is Haiku aiming to become a better OS, or is it 
going to aim for a minimal common denominator in the name of 
interoperability?

> We can't change the world by ignoring it.
>   

You are suggesting to be pragmatic and simply go with the status quo, 
but Haiku has never been about going with the status quo. Had we been 
pragmatic (as opposed to idealistic) as you suggest from the very 
beginning, Haiku would have not been started in the first place.

FWIW and IMVHO. Cheers,

Jorge
