Recently when I've been doing the build, I have just been moving
iconv.h to the Desktop temporarily, building, and then moving the
header back.  I know the plan is to do builds of GCC/BeZillaBrowser
and the like without linking in libiconv (and hence not including it
by the build system), so I'm guessing the idea though with the
--without-libiconv-prefix flag is just to cover the cases where
libiconv happens to be installed anyways.

I suppose that works better than the instructions saying, "If you have
iconv.h on your system, move it somewhere else first instead of using
the handy --without-libiconv-prefix flag".  :)

Oh yeah, and as for the PCH flag, I still use it too.  It seems to be
an mmap-related issue, but it seems like platforms that struggle with
it often make a host-specific file alongside their target files to
work around it instead of using --disable-libstdcxx-pch.  In our case,
I think using --disable-libstdcxx-pch is the much more pragmatic
solution though.
