>  mechanism used in BeOS, that is to press Control+Option-Tab (which didn't

Nice, this is a feature from Mac OS X I've always liked.

> * Did not change the window switch logic, though, so it's still not really

Well for what it is worth Mac OS X also just toggles through the
windows in order. I assume you want to mimic some of the "quick
switch" logic like for the Ctrl-Tab case (toggling between the same
two windows?) I don't know if that is necessary.

Also this breaks some of the keybindings for the Switcher/Twitcher
(BTW we should probably change the source code to match the name used
in documentation), which was supposed to use Ctrl-~ to go back in the
list (along with the left arrow key.) Though in my testing I don't
think it ever worked. Maybe Ctrl-1 would be another option for that
which is fairly close to the Tab key. This way you don't need your
other hand to use the arrow keys or do finger gymnastics holding Shift
to quickly go left in the app list.
