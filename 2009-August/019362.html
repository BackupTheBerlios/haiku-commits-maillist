"Wriite boot sector" does not install bootman... but bootman does
indeed work (I am using it on this netbook to chainload windows XP,
grub installed to my linux partition, and two copies of Haiku).

You just type "bootman" from a terminal to launch it. Caution must be
used, however, if one wants to replace grub with bootman, while still
retaining grub on their linux partition. You must boot to Linux and
install grub to a specific partition first, before using bootman to
overwrite the MBR. (i've been meaning to throw some documentation on
haiku-os.org about this process).

Unfortunately bootman has two issues: 1) it cannot be installed to any
drive other than the one you boot Haiku from (seems like an oversight
to me) and 2) it cannot boot a partition on any other disk than the
one it is installed to (I kinda understand this limitation, but it
would be nice to at least be able to chainload the MBR from another
drive entirely)
