On Thu, 16 Oct 2008 10:58:52 -0500, Rene Gollent wrote
> > Modified:
> >   haiku/trunk/headers/os/app/Message.h
> >   haiku/trunk/src/kits/app/Message.cpp
> > Log:
> > Introduce BMessage::CompareData() which is a Haiku API extension. It compares
> > the data of two BMessages and allows recursive comparison (BMessage inside
> > BMessage). Note that using this API might require you to recompile your app in
> > the path to R1.
> 
> Nice :) Do you think it'd be worth using that to implement
> operator==(const BMessage&) as well?

I had it this way first, but we decided not to implement a == operator. It
doesn't really seem to be too common to have that operator in Be classes that
are not of some basic type. Also this only compares data, i.e. not the what
field and other header data of the message, so using it for a == operator
could easily get you an non obvious result. It's really just about comparing
the contained data, not the message whole container itself, so it's not really
a perfect fit.
