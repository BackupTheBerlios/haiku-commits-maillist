> mmlr at mail.berlios.de wrote:
> > Log:
> > Changed the score calculation for type ahead in the Tracker. 
> 
> Before it would at least go to a close item if no matching item was 
> found. Does it still do that?

It didn't, it would select the topmost pose then, which is obviously 
not useful. Sorry about that, I changed it in r28234 to just not select 
anymore if there is no match. Since we do an incremental search, the 
last selected pose is the closest match we have, so just keeping it 
probably makes the most sense.

Feel free to change again if the behaviour doesn't make sense to you. I 
did so because the substring match wouldn't actually work previously if 
an item with the starting letter was present. I really think that if 
you have a substring that matches a longer part of a name it should be 
picked over one just starting with the same letter. If I type "monitor" 
in the settings directory for example, I'd say it makes more sense to 
match "ActivityMonitor settings" instead of the "Media" folder.
