Yes, I was thinking along the same lines. But Ingo has created that setgcc script and the build system creates some links and so on to support it. I didn't want to mess with that or make the Installer routine any more complicated (like by adding custom scripts to packages...). But if you can figure out a simple way to install GCC from a package, please go ahead! The installation time on my T60 is now shortly below 20 minutes. The unzipping towards the end is really very fast compared to the copy process before. So it would gain a lot if more packages were unzipped, like you say.
