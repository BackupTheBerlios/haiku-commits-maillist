Axel Dörfler wrote:

>> Someone makes a patch for a file that has some coding style (it does 
>> not 
>> matter which style) and follows that style. Are you suggesting that I 
>> should have told him to rewrite the entire file in his patch? or are 
>> you 
>> suggesting that the new code in the file should have followed a 
>> different style?
>>
>> The first option is completely unreasonable and the second one would 
>> even make things worse in the file.
> 
> It wouldn't make things worse in that file, it would improve its coding 
> style :-)

Yeah, except that when confronted with a decision between consistency 
and style, I will always go with consistency. The reason for this is 
very evident if you happen to work in different projects with different 
styles. You will not use the style you think is best but the one that is 
more consistent.

> Of course, that patch should ideally be accompanied with a further 
> patch to fix the remaining coding style issues.Introducing new code 
> with broken style just creates more work for the poor soul that will 
> eventually fix the style one day - which is completely unreasonable.

OTOH, until someone actually does that, the code will be an even greater 
mess than it is now and this does not help anyone reading it. My point 
is, if some of us decide to rewrite MDR to fit the style (which I intend 
to do at some point, component by component) it is fine. What is not 
fine nor reasonable is to reject a very useful patch on the grounds that 
it does not follow our coding guidelines when the file it touches does 
not follow the guidelines either.

In fact, this creates a conundrum. Right now I am rewriting IMAP and 
that will, of course, follow our coding guidelines. OTOH, I have several 
bugs to fix that were reported in the bug tracker and although I am 
willing to work on then, if I have to rewrite every file that I touch so 
it follows the style I will most definitely postpone the changes for 
when I have a lot more free time than I do now. In this case, what is 
more important? Getting the bugs fixed or making sure every file follows 
the coding style? (that was a rhetorical question, BTW. At least I have 
no doubts about what is more important).

-Bruno
