I think in some sense it is a short term solution, since it is solving
the problem of a slow copying of files during installation. But at the
same time it may be a smart way to go because reading a zip file and
then unzipping it is probably faster than copying the same files
(especially from a CD.) As a matter of fact the new Mac OS X Snow
Leopard release has most executables compressed for the same reason,
since even file accesses from hard drives are orders of magnitude
slower than the CPU. Of course this also saves disk space. I think
compressing applications and libraries is something we should consider
for the future as well.

On another note reading about Mac OS X and some of it's history has
made me realize we should not worry so much about making Haiku R1 so
perfect. It is better to get it out there and then incrementally
improve it. Otherwise we run the risk of fulfilling the jokes of
having an alpha release now, a beta in 2015, and R1 in 2020.
