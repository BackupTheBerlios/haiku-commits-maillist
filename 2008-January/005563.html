> Shouldn't the disk device API already return the partitions in order?
> And I guess even if not, it would be confusing if the order in the list
> wouldn't be alphabetically (ie. with 0_3 before 0_2 or something like

The disk device API returns the partitions in the order provided by the 
respective partitioning system. That's not necessarily the same as the 
order by disk offset. E.g. primary partition 1 will be listed before 
primary partition 2, even if the former has a greater disk offset.
