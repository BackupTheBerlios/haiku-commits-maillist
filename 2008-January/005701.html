On 2008-01-21 at 17:54:33 [+0100], Rene Gollent <anevilyak at gmail.com> wrote:
> > BTW, I can wholeheartedly recommend to any developer to enable the kernel
> > tracing feature. I keep it turned on all the time for syscalls, teams, and
> > signals, and having the history of all syscalls is really fun, 
> > particularly
> > when one encounters a bug that's hard to reproduce. The performance impact
> > for usual applications is below the error of measurement really.
> >
> 
> I'm curious, what's the easiest way to enable tracing? Is there a
> UserBuildConfig flag for it, or must one change a header define?

You can either change the macro definitions in 
headers/private/kernel/tracing_conf.h or define those macros in your 
UserBuildConfig ("TARGET_KERNEL_DEFINES += ENABLE_TRACING=1 SYSCALL_TRACING 
... ;"). In the latter case you should touch the header file whenever you 
change the settings, since otherwise jam won't rebuild anything.

If you have enough memory, you can also increase the tracing buffer size (I 
usually use 100 MB), since the default of 1 MB doesn't last that long.

CU, Ingo
