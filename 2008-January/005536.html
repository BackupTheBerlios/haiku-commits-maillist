On 2008-01-16 at 10:34:11 [+0100], Axel Dörfler <axeld at pinc-software.de> 
wrote:
> I'm not sure if unsetting the temporaries is a good idea - it might
> render the command history useless when using the variables, and is
> also not that convenient to use.

Since they are automatically set by commands, a variable might already have 
been overwritten by a later command, when you use the history. So, if you 
really want to keep a variable value around, you should assign it to a 
persistent variable.

> Maybe add a command to purge them manually instead, and only remove
> them when there is no space in the temporary array left?

Yeah, I was thinking about putting them in a queue sorted by last access. 
When the array is full, the least recently used variable would be 
overwritten.

> Also, can't we change the return type of parse_expression() to uint64?
> At least with x86 this should be binary compatible.

Good point.
