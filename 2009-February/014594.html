> The problem here is the AudioProducer node code in MediaPlayer. It originally stems from Be sample code, and Ingo and I already made it much more flexible than it used to be. I think changing it to support any number of channels should not be very hard at all. Then the channel converter can be removed from the pipeline that is constructed, but it's important that the remaining parts of the pipeline all handle any channel count. I believe this may already be the case.

In both FormatProposal and FormatSuggestionRequested the AudioProducer
returns it's preferred format which is 2 channel float 44100.

I assume that one or both of these methods needs to change to try
different formats.

Can you tell me where this negotiation comes from?

I would think that the first attempt to connect to the mixer would be
with the format that the decoder proposed and then if no connection is
possible retry with the format the AudioProducer could convert to
based on the abilities of the Converters it has.
