(gmail crashed while I was answering this, so, shortening the reply)

-The layout of the preflet makes sense after all, so I'll just remove
the keyboard tab
-BOutlineList seems good but I don't think ICU gives interesting infos
on the languages allowing to sort them.
-The languages will be sortable in some way but I wanted a simple
working base to test changing a language, signalling it to the
localeRoster and sending a broadcast BMessage to all apps to tell them
to reload their catalogs. Also, i'm not sure if I should just add move
up/down buttons to the list (making the alphabetical order pointless)
or put a second list and add 'enable/disable' butttons to move
languages from the disabled to the enabled list. Any idea on that ?
-ICU don't offer anything about languages. Actually they use the same
class, Locale, for representing a coutry (with or without language)
and a language. The only information you can get about a language
there is the two-letter ISO code and the readable name. We don't have
the keyword system yet, however, maybe that will bring some more info.
But as there is nothing like even the script direction, I think the
BLanguage class would only have a constructor (taking the 2 letter
code as a parameter) and one single method, getReadableName(). It
sounds a bit strange to have a class just for that. But it can be
done.
-I borrowed the date&time preferences from tracker. I'm converting it
to use the layout engine. But maybe there will be some changes to it
to better fit ICU API.
