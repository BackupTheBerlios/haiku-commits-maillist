If the new Pe package is built from the current trunk, it contains at least one visual glitch introduced by the parentheses highlighting, which is still on my virtual TODO list. Of course I wouldn't mind, if someone else felt the urge to fix it. :-) It is easily reproducible, e.g. with the following text:

{
	foo
}

When positioning the caret right after the "{" and hitting return, the top pixel row of the "foo" line after the text is drawn with the highlight color. Not particularly bad, but encountered from time to time and staining the otherwise pleasant editing experience a bit. :-)
