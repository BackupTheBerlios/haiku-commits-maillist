Stephan Assmus wrote:
> I think the real problem is that Firefox doesn't use the system font size 
> for it's internal 1em font size. For example, the Haiku website seems to be 
> exactly the system font size at 0.9em. 1.0em is indeed too large (i.e. 
> larger than the system font). But as I said, the problem should IMHO be 
> fixed in Firefox.

At work, I had a look at the user guide with IE7. It too shows very large fonts at 1em. I 
don't think we'll see fixing of browsers (if that's what it is) any time soon.

> Maybe you should just keep that change until Firefox or 
> another browser will use the system font as 1.0em. Maybe that's 
> problematic, though, for some other reason and browsers cannot do that 
> without breaking other things...

Personally, I use Alt+/-/= in Firefox a lot to in/decrease/reset the zoom. I recently, 
though I've been using it for *years*, discovered the option to "zoom text only". I was 
always annoyed that the graphics were resized unfiltered (or at all). Neat. Ryan and 
Maxime, take note, that's a must have (plus maybe an option to resize bilinear filtered). :)

But these work-arounds aren't something joe-user normally does. So maybe, to make a good 
first impression on a interested potential Haiku user, we should keep the 0.9em for now. 
And implement François suggestion of an alternative css, but with the "normal" size 1em.
