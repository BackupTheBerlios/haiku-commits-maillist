Four things:
1) there are exactly no blank lines between the license header and the 
header guard
2) if slab_private.h is intended to be used outside of the slab 
allocator, it should also be save to be called from within C code; 
'extern "C"' does not exist in C, and has to be guarded by #ifdef 
__cplusplus
3) if slab_private.h is indeed private, it shouldn't be in the kernel 
headers, but in src/.../slab/
4) please make always sure headers are self-containing; "size_t" is not 
a native C type
