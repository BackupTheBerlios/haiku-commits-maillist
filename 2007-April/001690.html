Ingo Weinhold wrote:

> src/bin/... is actually not the right place for test programs. We have a 
> separate subtree (src/tests) for those. This one would fit into 
> src/tests/kits/net or src/tests/bin.

Agreed so far. Tests should be moved to src/tests.

> Also generally please don't add test programs to the public image. You can 
> add locally whatever you like using your build/jam/UserBuildConfig. If you 
> need to add something to the public image temporarily to make it easier for

Why not? This isn't a public image after all. This is a development image,
that is usually used by developers, and not a public distribution.
Perhaps other developers want to run stress test programms too, to test
on different hardware, etc.

I think that putting (usefull) tests into the image is the right way to go.

Distributing the raw development image is violating a lot of licenses,
those people building and *distributing* their images are in a legal
jeopardy. 

I think we should not try to transform the development image into
any kind of distribution. Including as much as useful for development
should be the way to go.
