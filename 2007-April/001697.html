On 2007-04-19 at 20:54:11 [+0200], Marcus Overhagen 
<marcusoverhagen at arcor.de> wrote:
> Ingo Weinhold wrote:
> 
> > Also generally please don't add test programs to the public image. You can
> > add locally whatever you like using your build/jam/UserBuildConfig. If you
> > need to add something to the public image temporarily to make it easier 
> > for
> 
> Why not? This isn't a public image after all. This is a development image,
> that is usually used by developers, and not a public distribution.
> Perhaps other developers want to run stress test programms too, to test
> on different hardware, etc.
> 
> I think that putting (usefull) tests into the image is the right way to go.

As I wrote, I'm not opposed to put tests into the image in general. If 
networking is a hot spot ATM and feedback from testers, particularly output 
from certain test programs, is desired, then I'm perfectly fine with 
including them. But if every developer includes all his tests, then we'll end 
up with an image twice as big, including a lot of obsolete tests. E.g. I know 
how my UserBuildConfig looks like and I know that most of the stuff I put on 
my image is of little interest to others.

> Distributing the raw development image is violating a lot of licenses,
> those people building and *distributing* their images are in a legal
> jeopardy.

That's right. You mentioned it before and I replied in pretty much the same 
we I'm replying now: We *do* already distribute the image 
(http://factory.haiku-os.org/); either we stop doing this or fix the 
licensing problems. It's just that obviously noone cares enough.

> I think we should not try to transform the development image into
> any kind of distribution. Including as much as useful for development
> should be the way to go.

I fully agree, that we should favor development use. Which is exactly why I 
don't want to have every imaginable test on it, since that adds to everyone's 
turn-around times.

CU, Ingo
