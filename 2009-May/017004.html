Nonsensical as in they are flat-out wrong, or nonsensical in the fact
that they are not currently an issue?  GCC 4.4.0 has new rules on this
front.  Seeing as I am working on GCC 4.4 behind the scenes, some
issues are coming up.  The slight changes above is to address those.
Since we have Werror turned on for kernel code, GCC 4.4 will in turn
cause an error when it comes across code that has non-void functions
that don't return anything.  It also does not allow for #elseif
without an argument, as preprocessor directives are handled
differently.

Now, I realize we don't have GCC 4.4 currently as an option to
natively or cross-compile Haiku (to anybody but me, probably), so this
is currently a non-issue.  It will become an issue if we ever go
beyond GCC 4.3.3 though.

It can be addressed at a later time if so desired though, for sure.
