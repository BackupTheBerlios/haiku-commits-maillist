Em Dom, Maio 24, 2009 4:45 pm, Axel Dörfler escreveu:

>> SoundPlay has a feature like this but not as extensive as I described
>> as it just creates a normal playlist based on the query so no live
>> updates happen.
>
> Which sometimes would be what you want to have, in order to randomize
> the order of the files later - queries might not have the order you
> want to play the items in.

You don't need to use the files in the order returned by the query. Each
entry will be a normal entry in the (now live) playlist and its position
on it is irrelevant as long as it is possible to, given an entry_ref, find
the relevant playlist item. Obviously you can not save the file positions
(or you could, if this was stored somehow), but that's because you opted
to use a live playlist.  We could have a "Save as normal playlist" option
too, or something.

Point is, being able to have a MP3 query like "Best MP3 Files" and be able
to load it and automatically add/remove files to the current playlist by
simply changing relevant attributes is an awesome feature (in my opinion
at least), and a very good tech demo of what live queries can do.
