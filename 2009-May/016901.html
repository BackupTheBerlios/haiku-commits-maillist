While you are at it, how about implementing support for queries as 
playlists. With all that you did now (great work, BTW) it should not be 
difficult to implement it more or less like this:

1 - Somehow make it possible to load a query as a playlist. Could it be 
by drag and dropping the query icon to the playlist or by doing the 
query directly on it (I guess the first option is easiewr to 
implement).
2 - This would result in an internal live query being constructed by 
the playlist so that whenever a file starts/stops satisfying the query, 
it would be added/removed.
3 - Some indication that this playlist is a live one would be great 
too, as the user would not be able to drag files to/from the playlist 
as it is automatically generated.

SoundPlay has a feature like this but not as extensive as I described 
as it just creates a normal playlist based on the query so no live 
updates happen.
