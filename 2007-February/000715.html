Maybe I should have changed the name of the constant (it already
existed). SET_INITIAL_MODE_MSG is the new message constant for the
Undo button, so when you press ESC or the countdown ends your changes
will be reverted to the state before the user pressed Apply (which is
the original behavior, BTW). Only the Revert button in the main dialog
uses BUTTON_REVERT_MSG which resets to the state at program start.
