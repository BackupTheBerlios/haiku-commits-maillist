> > Er, and do we need write_link() at all?

> It's not that we would really *need* it, but I thought it was a good
> idea instead of needing to delete and recreate the link whenever you
> want to change its target.

I wouldn't consider this a very common usage pattern though (dunno what 
you're doing on your system, though ;-)), at least not common enough to be 
worth optimizing, IMHO. Besides AFAIK there isn't even a userland API for 
that yet. There are probably cases more worth optimizing, like recursive 
directory deletion/emptying, but I think we should keep the FS interface 
slim and only add stuff that is very promising with respect to performance 
or simplicity (like get_vnode_name() for instance).
