No, it doesn't. In editors set to 80 columns you have to scroll to see the 
excess chars. And terminals soft wrap those lines, which makes files with 
lots of too long line look terrible.

> Our coding style covers it like this which I find very reasonable:

Which I find really absurd. Most open source projects use an 80 columns 
limit (I actually don't know a single exception) and it's a natural limit 
(terminals), even under BeOS. Besides, it's also a typographical fact that 
the readability of text decreases beyond a certain line length (IIRC it's 
even < 70 chars; which is why newspapers use columns, BTW). This might not 
hold for source code to the same extend, but it definitely applies to 
comment passages.

Oh, and regarding the coding style: fs_interface.h had lines with 120+ 
chars.
