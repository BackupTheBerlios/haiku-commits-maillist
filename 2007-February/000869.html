Ah, sorry, my mistake. The "things to link against" parameter is a bit 
tricky, since it also allows you to specify library abbreviations like "be" 
which will then appear as "-lbe" in the linker line (for target 
r5/bone/dano at least, for haiku the name is first mapped to "libbe.so"). 
The responsible LinkAgainst rule tries to determine by looking at the given 
string whether it shall do that or treat it as a build system target. If 
the string has grist, other path components, or starts with "lib" it is 
considered a target, otherwise it ends up being prefixed with "-l". As a 
work-around for targets that are thusly mistreated, one can prepend 
"<nogrist>" as an indicator (the rule chops it off again).

Long story short: "<nogrist>app_server" it is. Sorry for the confusion.
