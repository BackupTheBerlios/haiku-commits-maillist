Hi devs,
 i wanted to leave a few words here so that those that would like to 
lurk or work with me know what i have in mind.

The motivation came a few month ago, in a discussion concerning a bug in 
Tracker with stippi:

> Alex:
> It's been fun digging old Tracker bugs, i'm more and more tempted to 
> simplify/factorize some parts of the code, but i think we should have at 
> least a working (well, less buggy) version without touching the structure 
> for now even if the temptation is great. I've left tons of comments in my 
> BPoseView that identifies code duplication and propose solutions. I think 
> that some parts could be replaced one by one by a more modular design, 
> without really changing the implementation yet, just decouple some 
> aspects and reuse most of the present code as the implementation 0. I'm 
> more for modularizing bit by bit by keeping it in a workable state, not 
> doing a full rewrite from scratch.

Well, i gave in to temptation :) doing a branch to make it smooth, i 
don't promise anything, we'll see how it goes, the first goal for now is 
to fix properly http://dev.haiku-os.org/ticket/2372 , by reworking how 
extent calculations are managed.
That might be all, but if i  ever continue on, i'll probably only touch 
BPose/BPoseView as there is enough to do there already!

See ya,
Alex
