I know where you're coming from. The 80 character limit indeed often 
makes the code harder to read; however, if you're not seeing the code 
at all (as it's beyond the window limits), it makes it much harder to 
read :-))
The advantage of the limit is that you can have two (or more) windows 
next to each other (like the header and the source) and can see 
everything at once. Originally, we had a limit between 80-90 
characters, and that got more strict over time. I also think we should 
have a fixed line limit; long lines are also not easy to read even with 
large editor windows. Whether it's 80 or 90 characters I don't really 
care though, both fits on screen well :-))
