On 2009-03-29 at 22:56:12 [+0200], Rene Gollent <anevilyak at gmail.com> wrote:
> On Sun, Mar 29, 2009 at 3:52 PM, Axel Dörfler <axeld at pinc-software.de> 
> wrote:
> > It's really not a good idea to use floating point computations for that
> > within the scheduler. Use fixed point integers instead if you need
> > this.
> 
> Actually integer precision should be good enough here, come to think
> of it. What's the problem with that particular code though? The
> denominator's fixed so it's not going to cause division by zero or any
> other floating point exception I'm aware of, which I'm assuming is the
> potential problem you're alluding to.

We do only save the floating point state on context switches not on 
interrupts. The scheduler can be invoked from interrupts (most notably the 
timer interrupt), so it really shouldn't perform any floating point 
operations. At least not without saving and restoring the state explicitly, 
which would be way too expensive in this case, though.
