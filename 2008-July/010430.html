> Is it generally desired to update the apps for this? I noticed that some 
> of the recent changes broke the bitmap arrow buttons used in Vision's 
> network prefs (they now draw a green background instead of transparent). 
> The drawing mode being used is B_OP_ALPHA w/ blending mode 
> B_ALPHA_OVERLAY, though it does appear to be using B_RGB32 bitmaps. This 
> did work in R5 and Dano though.

We had done some tests and decided that the BeOS way was inconsistent. The 
reason is that even B_RGB32 bitmaps can have B_TRANSPARENT_MAGIC_RGBA32 
pixels in them. On BeOS B_OP_OVER will regard the 
B_TRANSPARENT_MAGIC_RGBA32 pixels in B_RGB32 bitmaps, while it will draw 
B_RGBA32 bitmaps using the full alpha channel. B_OP_ALPHA on the other hand 
ignores B_TRANSPARENT_MAGIC_RGBA32 and treats them like B_RGBA32 bitmaps. 
Haiku is now more consistent now.

It is easy to change back though, we left the needed code commented out 
with an explaination. You can have a look at 
src/servers/app/drawing/Painter/Painter.cpp and experiement yourself if you 
like.
