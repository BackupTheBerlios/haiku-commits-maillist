One possible solution would be to switch to a different clipping solution 
when reaching a certain complexity of the region. A while back I started to 
experiement with a "BitfieldRegion". The performance of hittesting pixels 
would be independent of the complexity of the region, and operations like 
intersecting, excluding and including regions could be done "in hardware". 
The time would only depend on the area covered by the region(s). The method 
has of course some memory requirements as well as overhead when converting 
clipping rect regions into bitfield regions. The drawing code would have 
one more compare operation per pixel and more memory access. I can't really 
tell whether it's worth it or not, like I said it's not exactly common that 
clipping regions become so complex.
