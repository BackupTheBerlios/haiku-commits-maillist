Observations: The raw glyph rendering speed is 1.42 times faster in ZETA 
for B_OP_COPY. However, Haiku still manages to outperform ZETA by a factor 
of 2.88 for the duration to run this test. One possible explaination would 
be that ZETA actually limits screen updates to the screen refresh rate. 
This is something that Ingo previously proposed we should do as well. It 
would also explain the "bad" latency on ZETA between invoking Invalidate() 
and having Draw() be called as a result. All in all, the Haiku text 
rendering performance seems to be not too bad. Especially considering that 
Haiku uses 256 levels of anti-aliasing while ZETA only uses 16 (or so it 
seems when looking closer). Also encouraging is that the impact of subpixel 
anti-aliasing is so small, only 1.14 times slower. I would have expected 
much worse, kudos to Andrej!

In the next few days I would like to extend the benchmarking app to include 
other tests as well. Will keep you posted!
